# スタイリングルール

UI コンポーネントのスタイリングルールをまとめる。Tailwind CSS の利用を前提とする。クラスや中間表現を不要に抽出しない考え方は [コーディングルール](./コーディングルール.md) の「不要な抽出を避ける」と共通する。

## 1. className で Tailwind クラスを使う

### 方針

- スタイルは `className` への Tailwind クラス指定を優先する
- 例外として、実行時に値が決まる場合や要素に `className` を付けられない場合は `style` props や独自 CSS を使う

### 理由

- **指定方法を一本化できる**: スタイルの指定経路を `className` の Tailwind に揃えることで、要素のスタイルを 1 箇所で読み取れ、`style` props と CSS クラスのどちらを見るべきか迷わずに済む
- **デザインシステムの制約に乗せられる**: Tailwind のユーティリティはスペーシングやカラーなどの設計トークンに沿っており、任意値の散乱を防ぎながら一貫した見た目を保てる
- **静的に書けないものだけを例外にできる**: Tailwind はビルド時に確定する静的なクラスからしか CSS を生成しないため、実行時に決まる動的な値や `className` を付けられない領域はこの原則の対象外となる。例外をこの範囲に限ることで、静的に書けるスタイルが `style` props や独自 CSS に逃げるのを防げる

### 実装方法

```tsx
// 非推奨: style props や独自 CSS クラスで指定する
<div style={{ padding: 16 }} className="card">
  ...
</div>

// 推奨: className に Tailwind クラスを指定する
<div className="p-4 rounded-lg border">
  ...
</div>
```

## 2. クラスのマージには `cn` 関数を使う

### 方針

- 複数のクラスを結合・条件付与する際は `cn` 関数を使う
- テンプレートリテラルによるクラスのマージは避ける

### 理由

- **クラスの競合を解決できる**: `cn`（`clsx` + `tailwind-merge` 相当）は後勝ちで Tailwind クラスの衝突を解決するため、`p-2` と `p-4` のような競合を意図通りに上書きできる。テンプレートリテラルでは両方が残り、適用結果が不定になる
- **条件分岐を安全に書ける**: 条件付きクラスや `undefined` / `false` の混入を `cn` が吸収するため、テンプレートリテラルで生じがちな余分な空白や `undefined` の文字列化を避けられる

### 実装方法

```tsx
// 非推奨: テンプレートリテラルでマージする
<button className={`px-4 py-2 ${isActive ? "bg-primary" : ""}`}>...</button>

// 推奨: cn 関数でマージする
<button className={cn("px-4 py-2", isActive && "bg-primary")}>...</button>
```

## 3. デザインシステムのカラーを使う

### 方針

- カラーはデザインシステムで定義されたカラーを優先し、任意のカラー値の指定は控える
- 特別な意味や意図があって任意のカラーを使う場合は、その意図や理由をコメントで補足する

### 理由

- **テーマ変更に追従できる**: デザインシステムのカラートークンを参照しておけば、テーマやブランドカラーの変更が全体に一括で反映され、個別の修正が不要になる
- **逸脱を意図的なものに限定できる**: 任意カラーの使用にコメントを必須とすることで、トークンから外れる箇所が「意図したもの」だけに保たれ、レビュー時に逸脱の理由を確認できる
- **繰り返す表現はトークン化し、一点物は任意値でよい**: LP のようなデザイン性の高いページでも原則は変わらない。複数セクションで繰り返す色・グラデーションはトークン（または CSS 変数）に集約すれば一括で変更でき、そのページ・箇所でしか出ない一点物の色はコメント付きの任意値でよい。無理にすべてをトークン化すると命名と追跡のコストだけが増える

### 実装方法

```tsx
// 非推奨: 任意のカラー値を理由なく指定する
<span className="text-[#3b82f6]">...</span>

// 推奨: デザインシステムのカラーを使う
<span className="text-primary">...</span>

// 例外: 意図があって任意カラーを使う場合はコメントで理由を補足する
// 外部ブランド（X）の規定色に合わせる必要があるため
<span className="text-[#1da1f2]">...</span>
```

## 4. 共通デザインは React コンポーネントに切り出す

### 方針

- 複数箇所で使われるデザインの共通化は、CSS クラスではなく React コンポーネントとして切り出す

### 理由

- **Tailwind 公式が推奨する方法**: Tailwind は、React などのフレームワーク利用時に複数ファイルでスタイルを再利用する場合の「best strategy」としてコンポーネント化を挙げており、構造とスタイルを 1 箇所にカプセル化できる点を理由としている（[Tailwind CSS: Reusing styles](https://tailwindcss.com/docs/reusing-styles)）
- **責務と利用方法を型で表現できる**: React コンポーネントに切り出せば、props を通じてバリエーションや状態を型で表現でき、利用側は中身のクラス構成を知らずに済む。共通 CSS クラスでは適用条件やバリアントが暗黙になりやすい
- **変更の影響範囲を閉じ込められる**: 見た目の修正をコンポーネント内に閉じ込められるため、利用箇所すべてに一括で反映でき、クラスの貼り直し漏れを防げる

### 実装方法

```tsx
// 非推奨: 共通の見た目を CSS クラスで共有する
<span className="badge">新着</span>
<span className="badge">セール</span>

// 推奨: React コンポーネントとして切り出す
<Badge>新着</Badge>
<Badge>セール</Badge>
```

## 5. クラスは className に直接記載する

### 方針

- 中間変数や定数にクラス文字列を持たせることは避け、`className` に直接記載する
- 例外として、表示要素を配列で持ち、要素ごとに異なるクラスを指定したい場合は変数として持ってよい

### 理由

- **エディタ・linter の支援が追加設定なしで効く**: Tailwind CSS IntelliSense はデフォルトで `className` などの属性内のみを対象に補完・ホバー・クラス競合検知を行う。`className` に直接記載すれば、これらの支援が追加設定なしで効く。変数に抽出すると属性内ではなくなり、`tailwindCSS.classFunctions` や `experimental.classRegex` などを設定しない限り、補完や lint の対象外になる
- **追跡コストを増やさない**: クラス文字列を変数に抽出すると、`className` の参照箇所から定義箇所へ視線を往復させる必要が生じる。意味が自明なクラスを抽出しても、追跡コストだけが増える（[コーディングルール](./コーディングルール.md) の「不要な抽出を避ける」と同じ考え方）
- **配列での出し分けは抽出に価値がある**: 要素ごとに異なるクラスを配列で持つ場合は、データと表示の対応を一元管理するための抽出であり、重複や対応付けの明確化という目的を満たす

### 実装方法

```tsx
// 非推奨: クラス文字列を変数に抽出する（複数箇所で使うならコンポーネント化する）
const containerClass = "flex items-center gap-2";
return <div className={containerClass}>...</div>;

// 推奨: className に直接記載する
return <div className="flex items-center gap-2">...</div>;
```

```tsx
// 例外: 要素ごとに異なるクラスを配列で持つ場合は変数として持ってよい
const tabs = [
  { label: "概要", className: "text-primary" },
  { label: "詳細", className: "text-muted-foreground" },
];

return tabs.map((tab) => (
  <span key={tab.label} className={tab.className}>
    {tab.label}
  </span>
));
```
